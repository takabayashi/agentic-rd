03 Ingest flow — Wikipedia SSE built

Connect pulls the SSE stream, strips the data: prefix, fail-closes on heartbeats, filters to en.wikipedia article edits, and projects a clean schema.

sequenceDiagram
  participant W as Wikipedia SSE
  participant C as Connect http_client
  participant B as Bloblang transform
  participant L as LLM branch

  C->>W: GET stream with User-Agent
  W-->>C: text/event-stream frames
  loop each line
    C->>B: raw frame
    B->>B: keep data prefix lines only
    B->>B: parse_json catch deleted
    alt heartbeat or non-JSON
      B-->>C: drop fail-closed
    else valid edit
      B->>B: filter type edit bot false ns 0 en wiki
      B->>B: project schema + size_delta int64
      B->>B: epoch to ISO event_ts
      B->>L: clean edit record
    end
  end

04 Multi-step LLM reasoning loop

Pass 1 + escalation live in connect/classify.yaml (consumes wiki.edits.enriched).

sequenceDiagram
  participant E as Clean edit
  participant B1 as branch pass 1
  participant O as Ollama ollama_chat
  participant P as Robust parse
  participant S as switch confidence
  participant B2 as branch pass 2
  participant Sink as Sink topic + UPSERT

  E->>B1: request_map title comment size_delta
  B1->>O: chat completion JSON
  O-->>B1: raw model text
  B1->>P: extract first brace block
  P->>P: normalize label to enum
  P->>P: clamp confidence 0 to 1
  P-->>S: label + confidence
  alt confidence below threshold or unclear
    S->>B2: escalate richer context
    B2->>O: chat completion
    O-->>B2: raw model text
    B2->>P: reuse robust parse
    P-->>Sink: escalated true
  else high confidence
    S-->>Sink: escalated false
  end

05 Connect pipeline stages staged + live

Three services on compacted topics. Ingest projects the schema; enrich fetches the diff and gates on the schema_version contract (bad rows to dead-letter); classify runs pass-1 + escalation (shared Bloblang parse) and fans out to topic + Postgres UPSERT + audit. Each service exposes Prometheus metrics on :4195.

07 Domain model built

Stable contract in triage/models.py. Shared by the dashboard, JSON API, and future pipeline UPSERT. rev_id is the natural PK.

classDiagram
  class EditView {
    +int rev_id
    +string title
    +string editor
    +string comment
    +Label label
    +float confidence
    +bool escalated
    +int size_delta
    +string uri
    +datetime event_ts
    +datetime classified_at
  }
  class Label {
    vandalism
    substantive
    trivia
    unclear
  }
  class ClassifiedEditsTable {
    +BIGINT rev_id PK
    +CHECK label enum
    +CHECK confidence 0 to 1
    +INDEX event_ts DESC
  }
  EditView --> Label
  EditView .. ClassifiedEditsTable : maps to
  note for EditView "select_edits filters and sorts by confidence desc"

08 Service lifecycle and startup ordering

Compose gates services on healthchecks. Postgres applies db/init.sql on first boot; webapp and classify wait for a healthy Postgres. Webapp serves a 503 warm-up page when Postgres is down; /healthz is liveness-only, /readyz checks the DB.

stateDiagram-v2
  [*] --> Building
  Building --> Starting
  Starting --> WarmingUp
  WarmingUp --> Healthy: healthz 200
  WarmingUp --> WarmingUp: DB not ready
  Healthy --> Serving
  Serving --> Degraded: Postgres transient down
  Degraded --> Serving: pool recovers
  Serving --> [*]: compose down
